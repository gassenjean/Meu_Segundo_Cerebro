## Conceito (Síntese Própria)

O Framework iOS (Intelligence Operating System) é uma estrutura organizacional para times de agentes de IA, inspirada em cargos de desenvolvimento de software tradicionais. Em vez de prompts soltos, cada agente assume uma "persona de cargo" com responsabilidades limitadas e claras.

A estrutura permite criar "clusters" de trabalho onde você gerencia apenas o líder, e o líder gerencia os especialistas.

**Cargos Principais:**

1. **iOS Master:** O Orquestrador (ponto de contato).
2. **James:** Desenvolvedor Sênior (executa sem questionar).
3. **Whistle:** Arquiteto (planeja antes de codar).
4. **Kim/Queen:** QA (Quality Assurance - critica e bloqueia).
5. **Dave:** DevOps (infraestrutura).

---

## Princípio Central

> "Você não gerencia 200 agentes. Você gerencia 1 que gerencia 8." — Alan Nicolas

**O Princípio:** Hierarquia e Especialização. Agentes generalistas falham. Agentes especialistas com *jobs* definidos funcionam em escala.

---

## Aplicação ao Meu Contexto

### Para Desenvolvimento (KabaK)

- **Time de Produto:** Usar o iOS para criar o MVP do "Lovebook" ou Apps internos.
- **Fluxo:** Eu falo com o Master -> Master fala com Whistle (Arquiteto) -> Whistle gera PRD -> James (Dev) coda -> Kim (QA) valida.

### Para Conteúdo (Creator)

- **Adaptação dos Cargos:**
  - Whistle -> Editor Chefe (Planeja pauta)
  - James -> Redator (Escreve)
  - Kim -> Revisor (Verifica tom de voz e erros)
  - Dave -> Publicador (Sobe no WP/Instagram)

### Para Gestão (Gassen)

- **Delegação:** Nunca pedir "faça um site". Pedir ao Arquiteto "desenhe o site", aprovar, e então pedir ao Dev "implemente o desenho".
