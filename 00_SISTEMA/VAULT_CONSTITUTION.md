## PARTE 1: AS 3 LEIS FUNDAMENTAIS

### Lei 1: Fluidez

> "O sistema deve fluir. Se a organização dá trabalho, simplifique."

- Regra: Máximo 3 cliques para encontrar qualquer informação
- Regra: Se um processo tem mais de 5 passos, atomize
- Regra: Se você odeia fazer algo, automatize (após fazer 3x manualmente)

### Lei 2: Atomicidade

> "Notas pequenas (atômicas) são melhores que documentos gigantes."

- Regra: 1 Ideia = 1 Arquivo
- Regra: 1 Projeto = 1 Pasta
- Regra: 1 Conceito = 1 Nota

### Lei 3: Contexto é Rei

> "A IA só é genial se tiver acesso ao seu contexto."

- Regra: Toda nota deve ter frontmatter (created, updated)
- Regra: Toda nota deve pertencer a um MOC (zero órfãos)
- Regra: Todo arquivo deve ter localização correta

## PARTE 2: O MÉTODO 5C (Ciclo de Vida da Informação)

Todo conhecimento no vault passa por este ciclo:

```
CONSUMIR → CAPTURAR → CONECTAR → CRIAR → COMPARTILHAR
    ↑                                         |
    └─────────── FEEDBACK ←───────────────────┘
```

### C1: CONSUMIR (Input)

**Mentalidade:** Caçador

> "Entra na selva com alvo específico. Ignora ruído. Pega o tesouro. Sai rápido."

- Qualidade > Volume
- 1 ideia profunda > 100 reels rasos
- Bloco de tempo dedicado (não scroll infinito)

### C2: CAPTURAR (Save)

**Destino:** `_inbox/`

> "Salve rápido. Não organize agora. Só jogue na caixa de entrada."

- Não perca tempo formatando na captura
- Use extensões/atalhos para captura rápida
- Processe depois, em bloco dedicado

### C3: CONECTAR (Link)

**Mentalidade:** Curador

> "Como isso se conecta com o que eu já sei?"

**Critérios do Curador (só mantém se passar em pelo menos 2):**

| Critério | Pergunta |
| -------- | -------- |
| Inspirador | Me move emocionalmente? |
| Útil | Resolve um problema futuro? |
| Pessoal | É minha vivência única? |
| Surpreendente | Quebra meu modelo mental? |

- Linkar com notas existentes
- Adicionar a MOC relevante
- Se não conecta com nada = lixo ou arquivo

### C4: CRIAR (Output)

**Mentalidade:** Criador

> "O conhecimento só vive quando vira output."

- 1 Ideia = 1 Conteúdo
- Divergir (explorar) → Convergir (executar)
- Transforme notas em ativos (posts, produtos, projetos)

### C5: COMPARTILHAR (Ship)

> "O conhecimento só ganha valor quando sai do Vault."

- Publicar conteúdo
- Entregar projetos
- Ensinar outros

## PARTE 3: O MÉTODO MAPA (Orquestração de Agentes)

Como agentes e humanos trabalham juntos:

```
M (Mapear)   → Planejar sem IA. Desenhar fluxo. Definir outcome.
A (Atomizar) → Quebrar em tarefas ridículas de tão pequenas.
P (Programar)→ Escrever instruções/prompts para cada átomo.
A (Ativar)   → Soltar o agente. Não interromper (a menos que alucine).
```

### Regra Human-First

> "Nunca tente automatizar o que você não sabe fazer manualmente."

- Erro: Pedir para IA "fazer tudo" sem direção
- Correto: Você desenha a planta, IA assenta os tijolos
- Lei: Só automatize após fazer 3x manualmente e odiar

## PARTE 8: NOMENCLATURA (Lei de Nomes)

### Prefixos Obrigatórios

| Prefixo | Uso | Exemplo |
| ------- | --- | ------- |
| `MOC_` | Map of Content (sistema) | `MOC_Segundo_Cerebro.md` |
| `_MOC_` | Map of Content (categoria) | `_MOC_Projetos.md` |
| `TEMPLATE_` | Template | `TEMPLATE_Projeto.md` |
| `PROTOCOLO_` | Protocolo | `PROTOCOLO_Criacao_Arquivos.md` |
| `PLANO_` | Plano | `PLANO_Master_2026.md` |
| `STATUS_` | Status | `STATUS_Vault.md` |
| `CHECKLIST_` | Checklist | `CHECKLIST_Deploy.md` |
| `PROMPT_` | Prompt | `PROMPT_Nevoa.md` |

### Regras Gerais

- UPPERCASE para prefixos
- CamelCase ou Underscores para resto
- NUNCA espaços (usar `_`)
- Máximo 60 caracteres
- Sem caracteres especiais (exceto `_` e `-`)
