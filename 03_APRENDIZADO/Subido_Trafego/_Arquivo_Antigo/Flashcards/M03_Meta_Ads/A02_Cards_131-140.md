# Flashcards M03 A02 - Pixel e Eventos (131-140)

## Card 131
**Pergunta:** Por que o Pixel é chamado de "cérebro da conta de anúncios"?
**Resposta:** Porque ele afina e melhora a segmentação/direcionamento com base nos dados de comportamento que coleta, deixando o algoritmo cada vez mais preciso em encontrar pessoas similares.

## Card 132
**Pergunta:** Qual a diferença CRÍTICA entre conversão personalizada e eventos padrão/personalizados na criação de públicos?
**Resposta:** Conversões personalizadas NÃO conseguem criar públicos diretamente (só trackear URLs). Apenas eventos padrão e personalizados permitem criar públicos de remarketing.

## Card 133
**Pergunta:** Você tem um e-commerce no Shopify. Precisa implementar eventos manualmente?
**Resposta:** Não. Shopify (e outras plataformas como WooCommerce) já implementam eventos padrão automaticamente. Você só precisa adicionar o ID do Pixel.

## Card 134
**Pergunta:** Cliente quer trackear quantas pessoas scrollam até metade da página de vendas. Qual tipo de evento usar?
**Resposta:** Evento personalizado. Você criaria algo como "ScrolledTo50Percent" via Google Tag Manager ou código manual.

## Card 135
**Pergunta:** O que a Meta faz quando você define "AddToCart" como evento de otimização numa campanha?
**Resposta:** Treina o algoritmo para buscar pessoas com perfil similar a quem já adicionou produtos ao carrinho, aumentando chance de conversão.

## Card 136
**Pergunta:** Página de obrigado está em "/thank-you". Como criar evento sem mexer em código?
**Resposta:** Conversão personalizada baseada em URL. Configurar para disparar quando URL contém "/thank-you".

## Card 137
**Pergunta:** Três eventos padrão OBRIGATÓRIOS para qualquer e-commerce funcionar bem?
**Resposta:** 1) ViewContent (viu produto), 2) AddToCart (adicionou carrinho), 3) Purchase (comprou). Base mínima pra algoritmo aprender.

## Card 138
**Pergunta:** Por que Google Tag Manager é melhor que implementação manual de eventos?
**Resposta:** Interface visual (não precisa código), permite criar triggers complexos facilmente, centraliza todos tracking em um lugar, mais fácil manutenção.

## Card 139
**Pergunta:** Cliente B2B precisa trackear downloads de catálogo PDF. Evento padrão ou personalizado?
**Resposta:** Personalizado. Não existe evento padrão para isso. Criaria "DownloadedCatalog" ou similar.

## Card 140
**Pergunta:** Qual a real dificuldade de implementar Pixel + eventos básicos em 2024?
**Resposta:** Quase zero. Plataformas fazem automaticamente, GTM é visual, até manual é copy/paste. Pedro dramatiza complexidade que não existe mais.
