# Introdução à Requisição de API com Python

## Introdução

Este ebook aborda uma aula prática sobre como utilizar uma API para fazer requisições de dados usando Python. Vamos explorar os conceitos fundamentais de APIs, sua importância na comunicação entre sistemas, e demonstrar um exemplo prático de como obter informações de endereço a partir de um CEP utilizando a API do ViaCEP.

## O que é uma API?

API (Application Programming Interface) é uma interface que permite a comunicação entre diferentes plataformas e programas. Ela funciona como um contrato que estabelece como essa comunicação de dados deve ocorrer.

### Tipos de APIs:

- **APIs abertas**: Disponíveis publicamente, como as do IBGE e Google.
- **APIs fechadas**: Requerem autenticação, geralmente através de login e senha.
- **APIs pagas**: Exigem uma conta e possivelmente pagamento para acesso.

```
APIs são essenciais para a integração de sistemas e acesso a dados externos de forma estruturada e segura.
```

## Criando um Código Python para Requisição de API

Agora, vamos desenvolver um código em Python para fazer requisições à API do ViaCEP e processar os dados retornados.

### Por que usar Python?

Python é amplamente utilizado na área de dados devido à sua simplicidade e poder. É ideal para:

- Requisições de APIs
- Web scraping
- Análise de dados
- Criação de modelos preditivos

### Estrutura Básica do Código:

1. **Importação de bibliotecas**:
    
    ```python
    import requests
    ```
    
2. **Criação de lista de CEPs**:
    
    ```python
    lista_ceps = ["CEP1", "CEP2", "CEP3"]
    ```
    
3. **Inicialização da lista de endereços**:
    
    ```python
    enderecos = []
    ```
    
4. **Loop para requisição de cada CEP**:
    
    ```python
    for cep in lista_ceps:    url = f'https://viacep.com.br/ws/{cep}/json/'    response = requests.get(url, timeout=3)    endereco = response.json()    enderecos.append([        endereco.get('cep', ''),        endereco.get('logradouro', ''),        endereco.get('complemento', '')    ])
    ```
    
5. **Exibição dos resultados**:
    
    ```python
    for item in enderecos:    print(item)
    ```
    

### Explicação do Código:

- Utilizamos a biblioteca `requests` para fazer as requisições HTTP.
- Criamos uma lista de CEPs para consulta.
- Inicializamos uma lista vazia para armazenar os endereços.
- Iteramos sobre cada CEP, fazendo uma requisição à API.
- Utilizamos um `timeout` de 3 segundos para evitar gargalos.
- Extraímos apenas as informações desejadas (CEP, logradouro, complemento).
- Por fim, exibimos os resultados de forma estruturada.

## Dicas e Boas Práticas

1. **Tratamento de erros**: Implemente try/except para lidar com possíveis falhas na requisição.
    
2. **Rate limiting**: Respeite os limites de requisições da API para evitar bloqueios.
    
3. **Armazenamento de dados**: Considere salvar os resultados em um arquivo ou banco de dados para uso futuro.
    
4. **Parametrização**: Torne o código mais flexível, permitindo a entrada de CEPs pelo usuário.
    
5. **Documentação**: Comente seu código para facilitar a manutenção e compreensão.
