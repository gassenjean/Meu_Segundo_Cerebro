# Criando um Arquivo Executável com a Rio Engine

## Introdução

Neste guia, vamos explorar o processo de geração de um arquivo executável final usando a Rio Engine. Este é um passo crucial para disponibilizar seu projeto em computadores de clientes, shoppings, empresas ou indústrias. Vamos cobrir desde a exportação do projeto até considerações importantes sobre a instalação e execução do software resultante.

## Exportando o Projeto

### Preparação para Exportação

Antes de iniciar o processo de exportação, é importante ter seu projeto finalizado e pronto para distribuição. Certifique-se de que todas as funcionalidades estão implementadas e testadas.

### Processo de Exportação

Para gerar o arquivo executável, siga estes passos:

1. Na interface da Rio Engine, clique em "Platforms"
2. Selecione "Windows"
3. Escolha a opção "Package Project"

```
Dica: Crie uma pasta específica para armazenar o arquivo exportado. Isso ajudará na organização e facilitará a localização do executável final.
```

4. Selecione a pasta de destino para o arquivo exportado
   - Recomenda-se criar uma pasta chamada "Exportado" dentro do diretório do projeto
5. Confirme a seleção da pasta

### Tempo de Processamento

**Importante:** O processo de exportação pode ser demorado, variando de acordo com a complexidade do projeto e o desempenho do computador.

- Tempo estimado: 30 a 40 minutos
- Fatores que influenciam o tempo:
  - Tamanho do projeto
  - Velocidade do processador
  - Quantidade de memória RAM disponível

Durante este período, a Rio Engine estará compilando e empacotando todos os arquivos necessários para criar o executável standalone.

## O Arquivo Executável

### Localização e Execução

Após a conclusão do processo de exportação:

1. Navegue até a pasta selecionada como destino
2. Localize o arquivo executável gerado (geralmente com extensão .exe)
3. Para executar o software, basta dar um duplo clique no arquivo

### Características do Executável

- **Portabilidade:** O arquivo pode ser copiado e executado em outros computadores Windows
- **Interface:** Ao ser executado, o software carregará a interface do projeto, incluindo elementos como a "Vivian" mencionada na vertical

## Requisitos de Sistema e Instalação

### Dependência do .NET Framework

O executável gerado pela Rio Engine tem como requisito o .NET Framework. Isso significa que:

1. O .NET Framework deve estar instalado no computador do usuário final
2. Se não estiver presente, o software detectará automaticamente a falta

### Processo de Instalação Automática

Uma característica conveniente do executável gerado é sua capacidade de gerenciar a instalação do .NET Framework:

- Se o computador estiver conectado à internet, o software:
  1. Detectará a ausência do .NET Framework
  2. Baixará automaticamente os componentes necessários
  3. Instalará o .NET Framework sem intervenção manual do usuário

```
Observação: Este processo automático simplifica significativamente a distribuição e instalação do software para usuários finais.
```

### Considerações para Distribuição Offline

Para cenários onde o computador do usuário final não tem acesso à internet:

- Considere incluir os instaladores do .NET Framework junto com seu executável
- Forneça instruções claras para a instalação manual do .NET Framework, se necessário

## Conclusão

A geração de um arquivo executável usando a Rio Engine é um processo relativamente simples, mas que requer paciência devido ao tempo de processamento. O resultado é um software standalone que pode ser facilmente distribuído e executado em sistemas Windows.

Pontos-chave a lembrar:

- Planeje o tempo necessário para a exportação
- Organize bem os arquivos exportados
- Considere os requisitos do .NET Framework ao distribuir seu software

Com estas informações, você está pronto para exportar seu projeto e disponibilizá-lo para seus clientes ou usuários finais de forma eficiente e profissional.
